# Ví dụ cấu hình

Các ví dụ dưới đây phù hợp với lược đồ cấu hình hiện tại. Để tham khảo đầy đủ và ghi chú từng trường, hãy xem [Cấu hình](__OC_I19N_0000__).

## Bắt đầu nhanh

### Tối thiểu tuyệt đối

```json5
{
  agent: { workspace: "~/.openclaw/workspace" },
  channels: { whatsapp: { allowFrom: ["+15555550123"] } },
}
```

Save to `~/.openclaw/openclaw.json` and you can DM the bot from that number.

### Recommended starter

```json5
{
  identity: {
    name: "Clawd",
    theme: "helpful assistant",
    emoji: "🦞",
  },
  agent: {
    workspace: "~/.openclaw/workspace",
    model: { primary: "anthropic/claude-sonnet-4-5" },
  },
  channels: {
    whatsapp: {
      allowFrom: ["+15555550123"],
      groups: { "*": { requireMention: true } },
    },
  },
}
```

## Các mẫu phổ biến

### Thiết lập đa nền tảng

```json5
{
  agent: { workspace: "~/.openclaw/workspace" },
  channels: {
    whatsapp: { allowFrom: ["+15555550123"] },
    telegram: {
      enabled: true,
      botToken: "YOUR_TOKEN",
      allowFrom: ["123456789"],
    },
    discord: {
      enabled: true,
      token: "YOUR_TOKEN",
      dm: { allowFrom: ["123456789012345678"] },
    },
  },
}
```

### Secure DM mode (shared inbox / multi-user DMs)

If more than one person can DM your bot (multiple entries in `allowFrom`, pairing approvals for multiple people, or `dmPolicy: "open"`), enable **secure DM mode** so DMs from different senders don’t share one context by default:

```json5
{
  // Secure DM mode (recommended for multi-user or sensitive DM agents)
  session: { dmScope: "per-channel-peer" },

  channels: {
    // Example: WhatsApp multi-user inbox
    whatsapp: {
      dmPolicy: "allowlist",
      allowFrom: ["+15555550123", "+15555550124"],
    },

    // Example: Discord multi-user inbox
    discord: {
      enabled: true,
      token: "YOUR_DISCORD_BOT_TOKEN",
      dm: { enabled: true, allowFrom: ["123456789012345678", "987654321098765432"] },
    },
  },
}
```
Đối với Discord/Slack/Google Chat/MS Teams/Mattermost/IRC, ủy quyền người gửi mặc định là ưu tiên ID.
Chỉ bật khớp tên/email/biệt danh có thể thay đổi trực tiếp với `dangerouslyAllowNameMatching: true` của mỗi kênh nếu bạn chấp nhận rõ ràng rủi ro đó.

## Mẹo

- Nếu bạn đặt `dmPolicy: "open"`, danh sách `allowFrom` phù hợp phải bao gồm `"*"`.
- ID nhà cung cấp khác nhau (số điện thoại, ID người dùng, ID kênh). Sử dụng tài liệu của nhà cung cấp để xác nhận định dạng.
- Các phần tùy chọn để thêm sau: `web`, `browser`, `ui`, `discovery`, `canvasHost`, `talk`, `signal`, `imessage`.
- Xem [Nhà cung cấp](/providers) và [Khắc phục sự cố](/gateway/troubleshooting) để biết thêm ghi chú thiết lập chi tiết.
